# 因时而得谷（Yinshi）

“因时而得谷”是一个把飞书变成个人计算机上 Codex 远程控制面的项目。用户可以在手机或电脑端的飞书中向机器人发送任务，桥接服务把任务路由到指定机器上的本地 Codex thread，并把接收确认、中间进度、审批、最终文字、图片、文件和视频送回原飞书会话。

当前实现面向单一所有者控制自己拥有的设备，优先支持 Ubuntu；Windows 原生服务化仍在路线图中。

## 当前能力

- 一个 Codex 飞书机器人同时承担私聊控制面和群聊执行入口。
- 私聊用于新建/列出对话、查看额度、运行状态、临时无上下文任务和系统告警。
- 每个已登记 Codex thread 对应一个飞书群，群名带机器后缀。
- 持久化接收队列、发送队列、消息去重、断网重试和进程重启恢复。
- 在执行期间持续更新进度卡片，而非只在任务结束时回复。
- 暂存用户先发送的图片、视频、音频和文件，等待下一条文字要求后一起交给 Codex。
- 自动发送符合专用 outbox 约束、通过检查且未超过飞书限制的交付物。
- `/model`、`/fast`、`/permissions`、`/status` 与 `/compat` 飞书交互卡片。
- Codex CLI 升级后先关闭设置写入，通过真实 App Server 能力探测后再接受新版本。

## 多机器部署原则

当前架构要求“一台机器一个飞书企业自建应用”。不要让多台机器同时使用相同的 App ID/Secret：飞书长连接是集群随机投递而不是广播，同一应用的多个在线客户端会随机分走消息。

推荐命名：

```text
Codex-鼎盛笔记本ubuntu
Codex-鼎好主机4090
Codex-实验服务器01
```

每台机器分别保存自己的 App Secret、Codex 登录、本地 thread、桥数据库和配对信息。

## 项目状态

本仓库处于早期工程化阶段，已经在单用户 Ubuntu 主机上完成端到端运行验证，但尚未承诺稳定 API。进行生产、多人或公网暴露部署前，请先阅读安全文档并自行完成威胁建模。

本仓库尚未选定开源许可证。在许可证文件加入之前，代码公开可见不等于获得复制、修改或分发许可；未来正式开放贡献前需要由项目所有者明确选择许可证。
