# 安全说明

## 支持范围

当前版本面向“一个所有者控制自己拥有的设备”。它不是多租户远程执行平台，也没有经过面向不受信任用户的安全审计。

## 绝不能提交或分享的内容

- 飞书 App Secret
- `~/.codex/auth.json` 或系统凭据存储中的 Codex 令牌
- `bridge.sqlite`、WAL/SHM 文件
- inbox/outbox 中的用户附件
- 真实用户 open_id、union_id、user_id、chat_id
- API key、Cookie、SSH 私钥、云服务凭据

若凭据曾进入 Git 历史，应立即轮换凭据；仅删除最新文件并不能从历史中移除秘密。

## 默认部署边界

- 每台机器使用独立飞书应用。
- 只允许配对后的 owner 操作机器人。
- App Secret 只通过进程环境变量加载。
- 配置和状态目录限制为当前操作系统用户可读。
- 模型输出中的任意本机路径不会被无条件外发。
- 状态不明的 Codex `turn/start` 不自动重放。

## 权限风险

`danger-full-access + never`（YOLO）允许 Codex 在没有逐次批准的情况下访问整台机器。开源模板默认使用更保守的 `workspace-write + on-request`。只有设备所有者理解后果并确认机器、飞书账号和网络边界均可信时，才应显式启用 YOLO。

## 报告漏洞

仓库尚未建立公开安全邮箱。在正式开源前，请通过仓库所有者的私有联系方式报告，不要在公开 Issue 中包含漏洞利用细节或真实凭据。建立专用安全报告渠道是正式公开发布前的阻塞项。
